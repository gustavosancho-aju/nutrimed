## Story

**As a** nutrólogo e, por extensão, o paciente protegido pelo enquadramento regulatório,
**I want** disclaimers persistentes de "IA assiste, médico decide" presentes na UI base do produto,
**so that** fique inequívoco — em todo momento de uso e para fins de defesa CFM — que as contribuições do board são apoio à decisão, nunca conduta prescrita.

## Acceptance Criteria

1. O disclaimer "IA assiste, médico decide" está **presente e persistente** na UI base (ref. Critério de Aceitação 7 do E1 / FR19).
2. Existe o componente atômico `<DisclaimerNote>` com o texto "Sugestão de apoio. A conduta é sua." conforme `frontend-spec.md` §11 (component library) e §6 (mockup do card de contribuição).
3. O disclaimer é persistente: permanece visível no chrome base da aplicação (não é dispensável de forma que suma para sempre) e aparece no contexto das contribuições do board.
4. A linguagem segue o princípio de UX: tom de **sugestão** ("vale checar", "e se for…"), nunca de comando (frontend-spec §4.4) — a UI nunca empurra conduta.
5. O componente é acessível: contraste adequado, semântica correta e leitura por leitor de tela; respeita `prefers-reduced-motion` (sem animação obrigatória).
6. O disclaimer é reutilizável e consistente onde a IA se manifesta (chrome base + contribuições), sem duplicação divergente de texto.

### Princípios de UX que governam o texto (frontend-spec §4)

- §4.4: "**O médico é o decisor — a UI nunca empurra.** Linguagem de sugestão ('vale checar', 'e se for…'), nunca de comando. Coerente com 'IA assiste, nutrólogo decide'."
- §4.5: "Reduzir movimento é um direito" — `prefers-reduced-motion` desliga animação sem perda de informação.

### Contexto de compliance

- **FR19 / NFR10 / ADR-006:** o disclaimer é parte da postura CFM "apoio à decisão" — sustenta juridicamente o produto junto com a auditoria (Story 1.5). É **persistente** por design, não um aviso único dispensável.

### Fronteira de escopo

- Esta story entrega o **disclaimer base** no chrome e no card. As telas funcionais completas do board (feed, controles, Modo Foco, vídeo) são do **E7** — não construir aqui.
- O componente pode ser criado mesmo antes do board completo: vive no layout base e na biblioteca de componentes.

### O que NÃO fazer

- Não implementar o board, o feed de sugestões nem os controles (E7).
- Não criar lógica de consentimento (Story 1.4) nem auditoria (Story 1.5) — disclaimer é só UI.
