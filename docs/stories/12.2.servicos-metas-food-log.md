## Story

**As a** nutricionista que define metas e acompanha o consumo do paciente,
**I want** serviços em `@nutrimed/patients` para gravar/ler metas versionadas e registrar/agregar o consumo diário, sempre cifrados e auditados,
**so that** a UI e o bot leiam/escrevam esses dados no mesmo padrão seguro das medições do E11.

## Acceptance Criteria

1. `setNutritionGoal(db, patientId, setByUserId, effectiveFrom, values, key, origin?)`: **insere uma nova versão** (append, sem UPDATE destrutivo); `values_enc = encryptField(JSON.stringify({kcal,protein,carbs,fat}))`; escrita **auditada** (`writeAudit`/`auditedClinicalWrite`, `triggeredBy` default `'human-edit'`).
2. `loadCurrentNutritionGoal(db, patientId, key, asOf?)`: retorna a meta de **maior `effective_from ≤ asOf`** (default = hoje) decifrada, ou `null` se não houver.
3. `listNutritionGoalHistory(db, patientId, key)`: todas as versões decifradas, ordenadas por `effective_from`.
4. `addFoodLogEntry(db, patientId, input, key, origin?)`: grava `eaten_at`, `source`, `photo_ref`, `model_version` e `values_enc = encryptField(JSON.stringify({kcal,protein,carbs,fat,confidence,itemsLabel}))`; **auditado** (`triggeredBy` via `origin`, ex.: `'telegram-bot'`).
5. `listFoodLogByDay(db, patientId, dayISO, tzOffsetMinutes, key)`: entradas cujo `eaten_at` cai no dia (janela calculada pelo **offset explícito**), decifradas.
6. `sumFoodLogForDay(db, patientId, dayISO, tzOffsetMinutes, key)` → `DailyProgress { day, consumed:{kcal,protein,carbs,fat}, goal | null, remaining | null }`: soma os macros do dia e compara com `loadCurrentNutritionGoal`; **sem meta ⇒ `goal`/`remaining` = null** (não inventa meta — ADR-015).
7. Testes (espelham `patients.test.ts`): storage **ilegível em claro** (assert no `values_enc`); versão corrente por data; soma diária correta; janela por timezone; caso sem meta.
