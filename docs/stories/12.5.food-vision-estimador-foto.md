## Story

**As a** sistema que estima nutrientes a partir da foto do prato,
**I want** uma interface de estimativa plugável com uma implementação Claude (visão) e um fake,
**so that** a fonte de estimativa seja trocável (ADR-015/NFR8) e o fluxo opere sem credencial (degradação graciosa), no mesmo padrão do `@nutrimed/lab-import`.

## Acceptance Criteria

1. Pacote `@nutrimed/food-vision` expõe `IFoodEstimator.estimate({ base64, mediaType, filename? }) → FoodEstimate` com `{ values:{kcal,protein,carbs,fat}, confidence:'low'|'medium'|'high', itemsLabel?, notes? }`. **Nunca persiste.**
2. `sanitizeFoodEstimate(raw)`: aceita só nutrientes de `KNOWN_NUTRIENTS` (kcal/protein/carbs/fat), numéricos **finitos**, `confidence` válido; descarta o resto; **nunca lança** (entrada inválida ⇒ estimativa degradada). **Testado**.
3. `ClaudeFoodEstimator`: envia a imagem como **content block `image`** (`{type:'image', source:{type:'base64', media_type, data}}`) à Messages API (mesmo `fetch` do `@nutrimed/llm-anthropic`), pede JSON estrito; registra `modelVersion`; expõe `onUsage` para custo (NFR7).
4. `FakeFoodEstimator`: retorna estimativa **determinística** (testes e degradação).
5. `createFoodEstimator(env)`: `FOOD_ESTIMATOR=fake` força fake; `ANTHROPIC_API_KEY` → Claude; dev sem key → fake; **prod sem key → `null`**.
6. Sem persistência e sem UI nesta story. Testes: `sanitizeFoodEstimate` (campos/limites/`confidence`) + fake + factory.
