## Story

**As a** nutricionista,
**I want** vincular o Telegram do paciente (gerar código, ver status, revogar) e definir suas metas nutricionais na ficha,
**so that** eu controle o canal e as metas que o bot vai usar — mantendo o paciente sob meu escopo.

## Acceptance Criteria

1. Na ficha `/patients/[id]`, seção **"Canal Telegram"**: botão "Gerar código de vínculo" → server action chama `createPairingCode` → exibe o código **uma única vez** + instrução (`/start CÓDIGO`) + TTL visível + link do **termo de uso do paciente** (ADR-013 Decisão 5).
2. Status do canal: "não vinculado" / "ativo desde {data}" / "revogado"; botão **"Revogar canal"** quando ativo (chama `revokeChannel`).
3. Seção **"Metas nutricionais"**: form kcal/proteína/carbo/gordura + data de vigência → `setNutritionGoal`; exibe a **meta vigente** e o **histórico** (`loadCurrentNutritionGoal`/`listNutritionGoalHistory`).
4. **Escopo por médico dono**: usa `getCurrentUser()`; server actions rejeitam paciente que não pertence ao usuário (mesma proteção do E11).
5. Design premium reaproveitado (`card-premium`, `gold-hairline`, tokens `ink`/`brand`/`surface`).
6. **Verificação ao vivo** (browser): gerar código, definir meta, revogar canal — e ver cada estado refletir na ficha.
