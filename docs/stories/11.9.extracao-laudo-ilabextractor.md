## Story

**As a** sistema que importa laudos para pré-preencher medições,
**I want** uma interface de extração plugável com uma implementação Claude (PDF nativo) e um fake,
**so that** a fonte de extração seja trocável (canal reavaliável — ADR-012) e o fluxo opere mesmo sem credencial (degradação graciosa).

## Acceptance Criteria

1. Pacote `@nutrimed/lab-import` expõe `ILabExtractor.extract({ base64, filename? }) → ExtractedLaudo` com `{ kind: 'body'|'lab', measuredAt?, values: Record<campo,number>, notes? }`.
2. `sanitizeExtraction(raw)`: aceita só campos conhecidos por `kind` (bioimpedância: peso/massaMuscular/massaGordura/cintura/imc/pgc; exames: ldl/hba1c/insulina), descarta valores não-numéricos e chaves desconhecidas. **Testado**.
3. `ClaudeLabExtractor`: envia o PDF como **content block `document`** (base64) à Messages API (mesmo padrão `fetch` do `@nutrimed/llm-anthropic`), pede JSON estruturado; nunca persiste. Registra `modelVersion` para a auditoria da medição (NFR10).
4. `FakeLabExtractor`: retorna um rascunho determinístico (para testes e degradação graciosa).
5. `createLabExtractor(env)`: Claude se `ANTHROPIC_API_KEY` (ou `LAB_EXTRACTOR=fake` força o fake; sem key fora de produção → fake; produção sem key → `null` ⇒ UI cai p/ manual — NFR13).
6. Nenhuma escrita no banco nesta story (a persistência é a 11.10, sempre via confirmação médica — ADR-012).
