## Story

**As a** nutrólogo que acompanha vários pacientes,
**I want** a home como uma lista dos meus pacientes com acesso rápido a cada ficha e um caminho claro para iniciar uma nova consulta,
**so that** eu navegue pelo acompanhamento longitudinal em vez de só "abrir uma consulta solta".

## Acceptance Criteria

1. A home (`/`) lista os pacientes do médico logado (`listPatients`, escopo por `user_id`): nome, idade derivada (`computeAge`), objetivo e telefone quando houver.
2. Cada paciente da lista é um link para sua ficha `/patients/[id]`.
3. Há um CTA **"Nova consulta"** que leva ao fluxo de seleção/cadastro de paciente (rota `/consultations/new`).
4. O fluxo de seleção/cadastro + `startConsultationAction` (11.3) é **movido** da home para `/consultations/new`, sem perda de função (criar/selecionar paciente → consulta vinculada; gate de consentimento intacto).
5. Estado vazio tratado: sem pacientes, a home convida a iniciar a primeira consulta.
6. Design system existente (`card-premium`/tokens `ink`/`brand`/`surface`); navegação por teclado.
